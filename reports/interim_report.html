Interim Submission: The Automaton Auditor

Week 2 — Detective layer and partial StateGraph. This report covers architecture decisions, known gaps, and the planned flow for the judicial layer and synthesis engine.

1. Architecture decisions

1.1 Why Pydantic over dicts

  • Typed state and outputs: AgentState, Evidence, and JudicialOpinion are defined with Pydantic (BaseModel) or TypedDict so that parallel nodes cannot overwrite each other’s data by mistake. Reducers (operator.add for lists, operator.ior for dicts) are applied to annotated state keys, so detective outputs merge instead of replace.
  • Structured evidence: Evidence is a Pydantic model with goal, found, location, rationale, confidence, and optional content. This gives the Judges and Chief Justice a consistent, parseable structure and avoids freeform text as the only output.
  • Judicial opinions (planned): JudicialOpinion will be enforced via .with_structured_output() bound to the same Pydantic schema, so we avoid “Hallucination Liability” (rubric: Judges must not return freeform text).

1.2 How AST parsing was structured

  • No regex for structure: Repo forensic checks use Python’s ast module to parse src/graph.py and src/state.py. We walk the AST to detect StateGraph, add_edge, add_conditional_edges, add_node, and class bases (BaseModel, TypedDict).
  • Graph orchestration: analyze_graph_structure() collects all add_edge and add_conditional_edges calls, infers fan-out (multiple edges from the same source), and looks for a synchronization node by name (e.g. “aggregator”, “sync”, “collect”).
  • State and reducers: analyze_state_management() finds classes Evidence, JudicialOpinion, AgentState and checks for operator.add / operator.ior in Annotated type hints.
  • Sandbox check: check_sandboxed_tools() scans src/tools/ for tempfile/TemporaryDirectory, subprocess, and flags raw os.system.

1.3 Sandboxing strategy

  • Clone in temp dir: clone_repo(repo_url, target_dir) is designed to be called with tempfile.TemporaryDirectory() as target_dir. The node runs with tempfile.TemporaryDirectory() as tmp: clone_path = clone_repo(repo_url, tmp) so the clone lives only inside the context and is removed on exit.
  • No os.system: Git is invoked via subprocess.run() with capture_output=True, timeout, and return code checks. Repo URL is validated with a strict HTTPS GitHub pattern before clone.
  • Error handling: Clone failures (e.g. auth, network) raise; the RepoInvestigator node catches and returns a single Evidence with found=False and the error message.

2. Known gaps and plan for judicial layer and synthesis engine

2.1 Current gaps

  • Judges not implemented: Prosecutor, Defense, and Tech Lead nodes are not yet added; no dialectical opinions or criterion-by-criterion scores.
  • ChiefJusticeNode missing: No synthesis engine, conflict-resolution rules, or Markdown report generation.
  • VisionInspector missing: No image extraction from PDF or vision-model analysis (optional for execution per spec).
  • Structured output for Judges: When added, Judges must use .with_structured_output(JudicialOpinion) and retry on parse failure.

2.2 Concrete plan

  • Judicial layer: Add src/nodes/judges.py with three nodes (Prosecutor, Defense, Tech Lead). Each receives the aggregated evidence and rubric dimensions; each returns a list of JudicialOpinion (one per criterion) via an LLM bound to JudicialOpinion. Run all three in parallel (fan-out) from EvidenceAggregator, then fan-in to ChiefJustice.
  • Synthesis engine: Add src/nodes/justice.py with ChiefJusticeNode. Implement hardcoded rules: (1) Security override — Prosecutor-confirmed security flaw caps score at 3. (2) Fact supremacy — Defense claims overruled if Detective evidence contradicts. (3) Functionality weight — Tech Lead’s view weighted highest for “Graph Orchestration”. (4) If score variance > 2, require dissent summary and optional re-evaluation. Output: build AuditReport (Pydantic) and serialize to Markdown (Executive Summary → Criterion Breakdown → Remediation Plan).
  • Graph wiring: After EvidenceAggregator, add conditional edge to fan-out to the three Judges; add a second aggregator (or single “judge_aggregator”) node that collects opinions; then edge to ChiefJusticeNode → END. Persist final report to audit/ as Markdown.

State on edges: All edges carry full AgentState; detectives return only evidences (reducer operator.ior merges by source key).

Conditional edges and error handling (interim / planned):

NORMAL PATH: START ──► RepoInvestigator ──► EvidenceAggregator ──► END START ──► DocAnalyst ──► CONDITIONAL / ERROR PATHS (planned for final): RepoInvestigator ──► [clone/parse error] ──► return evidences: [{ found: false, rationale: "Error: ..." }] (still flows to EvidenceAggregator; no branch to END) DocAnalyst ──► [PDF missing/fail] ──► return evidences: [{ found: false }] EvidenceAggregator ──► [if any critical evidence missing] ──► optional: retry node or flag in state [else] ──► END (interim) or fan-out to Judges (final)

Error handling is node-internal: detectives catch exceptions and emit Evidence with found=False; conditional edges from EvidenceAggregator (e.g. “Evidence Missing” / “Node Failure”) will route to retry or a dedicated handler in the final graph.
